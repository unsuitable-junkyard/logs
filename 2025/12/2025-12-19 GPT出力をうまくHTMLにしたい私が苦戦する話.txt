## 🎯問題の本質

HTMLログ構造は見た目で改行してるように見えても、実際は…

<p>文1。</p> <p>文2。</p> <p>文3。</p>

これが全部、**「段落」じゃなくて「一文ごとの別pタグ」**になってる。
さらに <br> タグが混在して**一文内で改行も入ってくる**。

こうなると、**文の切れ目と段落の切れ目が不明瞭**になり、クリーナー処理やMarkdown変換時に「全部つながった文章」になるのよ。

## 🧨クリーナーや整形器が失敗する理由

## 1. <p> が段落じゃない

<p>＝段落ではなく、**文単位のタグ化**（自動音声読み上げや字幕向け仕様）

## 2. <br>が入ってるだけで段落風に見える

でも**視覚上の区切り**なだけで、論理的な段落ではない

## 3. <h2>や<hr>が段落ブロックを切らず、孤立してる

そのため、文章と文章の間の「意味のまとまり」が見えない

## ✅ 解決法：**段落判定のロジックを後付けする**

## ◉ おすすめ処理ステップ（擬似コード）

-
**<h2>〜</h2>までを見出しとしてマーク**

-
**連続する <p> タグをひとつの段落と見なして結合**

- 条件：次の<p>までに <hr> や <h2> が出なければ連結

-
<br> は無視（段落ではないが、文内改行ならそのまま残す）

-
最終的に「段落＝連続pブロック」「見出しはそのまま」で組み直し

## 🔧 Python用 BeautifulSoup＋正規表現コンボ（要望あれば実装出す）

擬似コード：

from bs4 import BeautifulSoup  def extract_paragraphs(html):     soup = BeautifulSoup(html, "html.parser")     output = []     buffer = []      for tag in soup.find_all(['h2', 'hr', 'p']):         if tag.name == 'h2':             if buffer:                 output.append(''.join(buffer))                 buffer = []             output.append(f"# {tag.text.strip()}")         elif tag.name == 'hr':             if buffer:                 output.append(''.join(buffer))                 buffer = []             output.append("---")         elif tag.name == 'p':             buffer.append(tag.text.strip() + '\n')  # ← 改行で擬似的に段落風に      if buffer:         output.append(''.join(buffer))      return "\n\n".join(output)
